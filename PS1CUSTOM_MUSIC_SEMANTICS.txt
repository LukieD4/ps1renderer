VAB+SEQ music player for the ps1renderer engine — INTEGRATED
==================================================================
(Originally delivered as a drop-in; everything below is now merged and
live in the tree. This doc describes the module as it exists today.)

FILES
  music.h / music.c   The sequencer + bank loader (PSn00bSDK 0.24 ships no
                      libsnd, so this module IS the sequencer).
  sound.h / sound.c   Existing module, extended for music support:
                        - spu_alloc() renamed/made public as sound_spu_reserve(),
                          plus a LIFO sound_spu_release() (used by music_unload).
                        - wind loop-scan matches (flag & 0x03) == 0x03 so a
                          sample that self-loops on its final block (flag 0x07)
                          still reports LOOP=Y.
                      Nothing else changed; wind behavior is identical.

INTEGRATION (main.c) — done
  1. init() calls, after sound_init():
         if (music_load("\\SONG.VAB;1", "\\SONG.SEQ;1"))
             music_play();
  2. main() calls music_tick() once per displayed frame, right after
     display() returns from its VSync.
  3. debug_text() prints, next to the SPU RAM line:
         FntPrint(-1, "MUSIC=%s LOADED=%s V=%d/%d BANK=%dB\n",
             music_playing() ? "ON" : "OFF",
             music_loaded() ? "Y" : "N",
             music_voices_active(), MUSIC_VOICE_COUNT,
             music_spu_used());

ISO — done
  iso.xml carries, alongside WIND.VAG:
      <file name="SONG.VAB" type="data" source=".../sound/music/SONG.VAB"/>
      <file name="SONG.SEQ" type="data" source=".../sound/music/SONG.SEQ"/>
  (Export both from OST Studio; the internal song name becomes the filename —
  keep it 8.3-friendly for the ISO. Drop new exports into sound/music/.)

PAL
  #define MUSIC_VSYNC_HZ 50 in music.h. Tempo is anchored to REAL vblank
  counts (VSync(-1) deltas) against the true centihertz rate — 59.94 Hz
  NTSC / 50.00 PAL, see MUSIC_VSYNC_CHZ in music.c — so a renderer that
  skips vblanks doesn't slow the music, and 60-vs-59.94 drift is gone.

VOICE MAP (compile-time partition, extend in one place)
  0        wind        (sound.c)
  1..7     future SFX  (one-shot barks/caws module)
  8..23    music       (music.c, MUSIC_VOICE_FIRST/COUNT)

STAGE-ASSET ROADMAP (future-proofing notes)
  Target shape: each stage ships a manifest listing its assets —
    MUSIC   one VAB+SEQ pair            -> music_load()/music_play()
    SFXBANK one-shot VAGs               -> future sfx_load_bank(): same
            (standalone, attached to a     chunked-CdRead + sound_spu_reserve
            character, or event-fired)     pattern as music.c's load_vab()
    CHAR    meshes/anims + attached sfx refs
    FX      particle definitions
  What this module already guarantees for that world:
    - data-driven paths, no hardcoded filenames;
    - load/play split so a stage loader can prefetch behind a fade;
    - one SPU budget (sound_spu_reserve) shared by all audio modules, so the
      manifest loader can check music+sfx fit BEFORE streaming anything;
    - teardown story: release in reverse load order (LIFO). If stages later
      need arbitrary unload order, upgrade sound.c's bump allocator to a free
      list — callers don't change.
  Suggested load order per stage: SFX banks first (long-lived), music last
  (swapped most often), so music_unload() can always LIFO-release cleanly.

CAVEATS (v1, by design)
  - One tone per program (matches the exporter). Multi-tone key-splits would
    need the tone-range scan added to note_on().
  - ADSR words are written verbatim from the VH; the exporter's UI→ADSR
    mapping is heuristic, so expect to calibrate envelopes against hardware
    once, then bake corrections into the exporter.
  - Pitch bend / vibrato / CC volume events are skipped (exporter never emits
    them yet).
  - VH must fit one 32 KB chunk: max 48 programs (MUSIC_MAX_PROGRAMS).
